Landing Page Bootstrap Rapi & Anti “Geser”

Checklist praktis agar layout stabil di semua device: container yang benar, overflow-x aman, gambar responsif, dan grid yang rapi.

24 Jan 2026 Muhammad Affizar Web Development

TOC Otomatis (H2/H3)

    Pernah bikin landing page, tapi begitu dibuka di HP tiba-tiba ada “geser” ke samping? Biasanya ini karena ada elemen yang melebar melebihi viewport.

    “Layout rapi itu soal konsistensi: lebar, jarak, dan grid.”

    1. Pakai Container dengan Benar

    Pakai .container untuk menjaga lebar konten konsisten. Untuk section full-width, tetap taruh .container di dalam section.

    Checklist container

    • Section full-width boleh, tapi kontennya dibungkus container
    • Hindari elemen besar “keluar” dari container

    2. Kunci Overflow-X (Anti Geser Paling Cepat)

    Kunci overflow-x pada html dan body untuk mencegah scroll horizontal muncul.

    Penyebab umum overflow

    • Gambar tidak responsif
    • Elemen absolute keluar layar
    • Row tanpa container (gutter keluar)
    • Lebar fixed terlalu besar

    3. Pastikan Semua Gambar Responsif

    Untuk Bootstrap, biasakan semua gambar konten memakai .img-fluid.

    Checklist gambar

    • class="img-fluid"
    • Gunakan object-fit: cover untuk thumbnail
    • Hindari width pixel besar tanpa max-width

    4. Rapikan Grid dan Row

    Struktur aman: container → row → col. Kesalahan kecil di grid bisa bikin layout “geser”.

    Pola aman

    • Row hanya untuk grid
    • Col selalu di dalam row
    • Jangan taruh komponen besar di luar container tanpa sengaja

    FAQ

    Karena ada elemen melebar melebihi viewport: gambar tidak responsif, grid salah, atau elemen absolute keluar layar.

    Buka DevTools lalu cek elemen yang lebarnya melebihi viewport. Bisa juga pakai outline sementara.

    Kesimpulan

    Kunci landing page anti “geser”: struktur container benar, grid rapi, gambar responsif, dan overflow-x aman.
