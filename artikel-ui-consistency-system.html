UI Consistency: Skala Spacing & Komponen Biar Konsisten

Framework kecil biar UI kamu konsisten: spacing scale, typography scale, komponen reusable, dan checklist implementasi supaya tampilan rapi di semua halaman.

24 Jan 2026 Muhammad Affizar UI
UI 7 menit baca Spacing + Komponen

UI yang “enak dilihat” itu sering bukan karena efek keren, tapi karena konsisten: jarak rapi, ukuran teks terukur, dan komponen dipakai berulang dengan pola yang sama. Kalau setiap halaman beda-beda aturan, user bakal merasa situs kamu “random”.

“Konsistensi bukan bikin desain kaku, tapi bikin pengguna nyaman karena polanya bisa ditebak.”

1. Pakai Skala Spacing (Jangan Ngira-ngira)

Daripada pakai padding/margin random (13px, 19px, 27px), mending pakai skala tetap. Pilihan aman: kelipatan 4 atau 8.

Contoh skala spacing

  • 4-based: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40
  • 8-based: 8 / 16 / 24 / 32 / 40 / 48 / 56 / 64

Aturan cepat

  • Jarak antar elemen kecil: 8–12
  • Jarak antar section: 24–40
  • Jarak antar card/grid: 12–24

2. Typographic Scale: Ukuran Teks Jangan Lompat-lompat

Setelah spacing, yang bikin UI berantakan biasanya font size yang nggak terstruktur. Buat “tangga” ukuran teks supaya hierarki jelas.

Contoh tangga teks yang aman

  • 12 label kecil / helper text
  • 14 body text (paling sering)
  • 16 body besar / highlight
  • 20 sub-judul section
  • 28–44 judul halaman (seperti di hero kamu)

3. Komponen Reusable: Button/Card/Input Itu Harus Sama

Konsistensi paling cepat kelihatan dari komponen. Kalau button di halaman A rounded 999, tapi di halaman B rounded 6, user langsung sadar “kok beda?”. Solusinya: tetapkan komponen inti dan variasinya.

Komponen inti yang wajib distandarkan

  • Button: primary / secondary / outline + size sm/md/lg
  • Card: border, radius, shadow, padding
  • Input: tinggi, border, placeholder, focus state
  • Badge/Pill: warna, border, font weight

Aturan variasi yang simpel

  • Kalau ada 1 style baru, tanyakan: “ini variasi atau komponen baru?”
  • Kalau cuma beda warna/ukuran: jadikan variant, bukan komponen baru
  • Nama class harus konsisten (mis: .btn-pill, .badge-soft)

4. Alignment & Grid: Konsisten Itu Juga Soal “Garis”

UI kelihatan rapi karena banyak elemen “jatuh” ke garis yang sama. Pakai grid/kolom (Bootstrap) dan atur konsistensi lebar container.

Checklist alignment

  • Lebar konten konsisten: container / max-width
  • Gutter grid seragam (mis: g-3 atau g-4)
  • Icon dan text align (baseline/center) di tombol & badge

5. Checklist Cepat UI Consistency

  • Spacing: pakai skala tetap (4/8-based)
  • Typography: tangga ukuran + line-height konsisten
  • Komponen: button/card/input seragam + variant jelas
  • Grid: container & gutter konsisten
  • Review: cek 3 halaman berbeda, harus “satu keluarga”

FAQ

UI consistency adalah keseragaman pola tampilan: spacing, typography, komponen, dan alignment agar semua halaman terasa satu sistem.

Gunakan skala kelipatan 4 atau 8 (mis. 4/8/12/16/24/32/40). Dengan angka tetap, jarak lebih konsisten dan gampang diaudit.

Mulai dari komponen inti (button, card, input), tentukan variant + size, lalu pakai class yang sama di semua halaman. Dokumentasikan aturan pakai + contoh.

Kesimpulan

UI yang konsisten itu hasil dari “aturan kecil” yang dipakai terus: skala spacing, tangga typography, dan komponen reusable. Sekali kamu rapikan sistemnya, bikin halaman baru jadi cepat dan hasilnya tetap seragam.
