UI Implementation: Figma → HTML/CSS Presisi

Cara menjaga implementasi agar mirip referensi: mulai dari spacing scale, typography scale, komponen modular, hingga checklist review.

24 Jan 2026 Muhammad Affizar UI Implementation

TOC Otomatis (H2/H3)

    “Kok hasilnya beda sama Figma ya?” Ini problem yang paling sering muncul saat UI implementation. Kuncinya bukan sekadar “rapihin CSS”, tapi bikin sistem yang konsisten: spacing, typography, dan komponen.

    “Figma itu blueprint. Implementasi yang rapi adalah soal mengikuti scale—bukan menebak-nebak ukuran.”

    1. Mulai dari Grid & Spacing Scale

    Jangan langsung styling detail. Pertama pastikan layout “kerangka”-nya benar: grid, padding section, dan jarak antar komponen.

    Spacing scale yang aman

    • Pakai kelipatan 4 atau 8 (4, 8, 12, 16, 24, 32, 40...)
    • Jangan campur jarak random (misal 19px, 27px) tanpa alasan
    • Section spacing lebih besar dibanding spacing antar elemen

    Contoh token spacing (praktis)

    :root{
  --sp-1: 4px;
  --sp-2: 8px;
  --sp-3: 12px;
  --sp-4: 16px;
  --sp-5: 24px;
  --sp-6: 32px;
  --sp-7: 40px;
}

.section{ padding: var(--sp-6) 0; }
.card{ padding: var(--sp-4); }
.stack-12 > * + *{ margin-top: var(--sp-3); }

    2. Typography Scale & Line-height

    Perbedaan yang paling terlihat biasanya ada di typography: font-size, line-height, dan font-weight.

    Checklist typography

    • Gunakan skala (misal 12, 14, 16, 20, 24, 32, 40)
    • Pastikan line-height sesuai (umumnya 1.4–1.8 untuk body)
    • Batasi variasi font-weight (misal 400/600/800 saja)

    Contoh token typography

    :root{
  --fs-12: 12px;
  --fs-14: 14px;
  --fs-16: 16px;
  --fs-20: 20px;
  --fs-32: 32px;
}

.text-sm{ font-size: var(--fs-12); line-height: 1.6; }
.text-base{ font-size: var(--fs-14); line-height: 1.9; }
.text-lead{ font-size: var(--fs-16); line-height: 1.8; }
.h2{ font-size: var(--fs-20); font-weight: 800; }
.h1{ font-size: clamp(28px,3.4vw,44px); font-weight: 800; line-height: 1.1; }

    3. Bangun Komponen Modular

    Kalau kamu bikin tombol, card, input, dan badge dengan pola yang konsisten, 70% UI akan terlihat “nyambung” tanpa effort besar.

    Komponen minimum

    • Button (primary/secondary/outline)
    • Card
    • Badge/Chip
    • Input + helper text

    Contoh komponen (1 style, banyak variant)

    .btn-ui{
  border-radius: 999px;
  padding: .55rem 1.05rem;
  font-weight: 800;
  border: 1px solid rgba(90,15,29,.12);
  transition: transform .15s ease, box-shadow .15s ease;
}
.btn-ui:hover{ transform: translateY(-1px); }

.btn-ui.primary{
  background: linear-gradient(180deg, var(--accent-2) 0%, var(--accent) 100%);
  color:#2a1a00;
  border-color: rgba(212,175,55,.55);
}

.btn-ui.outline{
  background:#fff;
  color:#241012;
}

.card-ui{
  border:1px solid rgba(90,15,29,.10);
  border-radius:18px;
  background:#fff;
  box-shadow: var(--soft-shadow);
  padding:16px;
}

    4. Workflow Review: “Mirip Referensi”

    Setelah implementasi 1 section, lakukan review singkat: cek jarak, alignment, ukuran font, dan radius.

    Checklist review cepat

    • Alignment: kiri/kanan/center konsisten
    • Spacing: konsisten (4/8 scale)
    • Typography: size + line-height cocok
    • Komponen: style sama di semua tempat

    Workflow 10 menit (praktis)

    • Menit 1–2: set container + grid
    • Menit 3–5: taruh spacing (padding/margin utama)
    • Menit 6–7: typography (h1/h2/body)
    • Menit 8–10: komponen (button/card/input) + cek ulang

    FAQ

    Biasanya karena scale typography tidak konsisten, spacing tidak memakai sistem, komponen tidak modular, atau token (radius/padding) tidak seragam.

    Mulai dari grid & spacing scale → typography scale → komponen. Setelah itu, review per section (spacing + alignment + line-height).

    Tidak wajib untuk proyek kecil, tapi minimal punya token sederhana: spacing, font size, radius, dan warna supaya konsisten.

    Kesimpulan

    UI presisi bukan soal “banyak CSS”, tapi soal sistem: spacing scale, typography scale, komponen modular, dan review rutin. Kalau kamu konsisten, hasil implementasi akan terlihat profesional dan mirip referensi.
