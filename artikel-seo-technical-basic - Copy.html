SEO Technical Dasar: Sitemap, Robots, dan Performance

SEO bukan cuma meta. Ada bagian “technical” yang bikin website kamu mudah diindeks dan cepat: sitemap, robots.txt, canonical, dan optimasi performance.

    Kalau on-page itu soal konten dan struktur, SEO technical itu soal “mesin”: apakah crawler mudah menemukan halaman, apakah halaman cepat, dan apakah tidak ada sinyal yang membingungkan.

    “Konten bagus tetap bisa kalah kalau web-nya lambat dan sulit diindeks.”

    1. Sitemap.xml: Peta Halaman untuk Crawler

    Sitemap berisi daftar URL yang ingin kamu indeks. Untuk website portofolio yang punya banyak artikel, sitemap membantu mesin pencari menemukan halaman baru lebih cepat.

    Contoh sitemap sederhana

    <?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://domainkamu.com/</loc>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://domainkamu.com/artikel.html</loc>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
</urlset>

    Minimal yang perlu kamu lakukan

    • Buat file sitemap.xml di root website
    • Isi URL penting (artikel, halaman utama, about, dll)
    • Submit ke Google Search Console (kalau sudah online)

    2. Robots.txt: Arahan Crawl yang Benar

    robots.txt itu “aturan main” untuk bot: mana yang boleh di-crawl, mana yang nggak. Ini bukan keamanan—jadi jangan simpan data sensitif lalu berharap robots menutup akses.

    Contoh robots.txt aman

    User-agent: *
Allow: /

Sitemap: https://domainkamu.com/sitemap.xml

    Kesalahan yang sering kejadian

    • Tanpa sadar memblokir seluruh website: Disallow: /
    • Memblokir folder aset penting (CSS/JS) → tampilan bot jadi “rusak”
    • Lupa update robots ketika pindah struktur folder

    3. Canonical: Hindari Duplikasi URL

    Kadang 1 halaman bisa diakses dari beberapa URL (mis. parameter, versi www/non-www). Canonical memberitahu mesin pencari “URL utama yang dianggap resmi”.

    Contoh canonical

    <link rel="canonical" href="https://domainkamu.com/artikel-seo-technical.html">

    4. Performance: Core Web Vitals (LCP, INP, CLS)

    Performance ngaruh ke UX dan bisa mempengaruhi SEO. Fokus ke 3 metrik utama: LCP (loading konten besar), INP (respons interaksi), dan CLS (layout geser).

    Quick wins yang paling sering efektif

    • Optimasi gambar: kompres, ukuran tepat, gunakan loading="lazy"
    • Hindari layout geser: gambar/iframe kasih ukuran (width/height) atau wrapper stabil
    • Kurangi script berat: minim plugin/library yang tidak perlu
    • Cache aset: set caching di hosting (kalau sudah deploy)

    Contoh gambar responsif (anti CLS)

    <img
  src="img/cover.webp"
  alt="Cover artikel"
  width="1200"
  height="630"
  loading="lazy"
  class="img-fluid rounded-4">

    5. Checklist SEO Technical untuk Portofolio

    • Sitemap: ada dan berisi URL penting
    • Robots: tidak memblokir halaman penting + set sitemap
    • Canonical: konsisten untuk URL utama
    • Images: kompres + ukuran benar + lazy load
    • Layout: minim CLS (anti “geser”)
    • Speed: minim script berat, gunakan cache di hosting

    Kesimpulan

    SEO technical itu fondasi: sitemap + robots membantu indexing, canonical mencegah sinyal duplikat, dan performance bikin user betah. Kalau fondasi ini beres, konten kamu punya peluang lebih besar naik.
