SEO On-Page untuk Portofolio: Meta, Heading, dan Internal Link

Cara bikin halaman portofolio kamu lebih mudah dipahami mesin pencari dan lebih menarik diklik: dari meta, struktur heading, sampai internal link dan schema.

24 Jan 2026 Muhammad Affizar SEO On-Page

TOC Otomatis (H2/H3)

    SEO on-page itu bukan “trik”, tapi cara bikin halaman kamu jelas. Mesin pencari butuh sinyal: halaman ini tentang apa, bagian mana yang penting, dan ke halaman mana harus diarahkan. Kalau rapi, peluang naik ranking dan CTR ikut naik.

    “On-page itu membuat halaman mudah dipahami: untuk user dan untuk mesin.”

    1. Meta Title: Judul yang Menjual + Relevan

    Meta title muncul di tab browser dan hasil pencarian. Buat yang singkat, jelas, dan ada kata kunci utama. Untuk portofolio, tambahkan konteks (mis. “Portofolio Web Developer”) atau nama.

    Contoh meta title yang aman

    <title>Portofolio Web Developer Bootstrap — Muhammad Affizar</title>

    Tips cepat

    • Taruh keyword penting di depan
    • Jangan terlalu panjang (hindari kepotong)
    • Jangan semua halaman pakai judul yang sama

    2. Meta Description: Biar CTR Naik

    Meta description itu “iklan mini” di Google. Walau bukan faktor ranking langsung, description yang bagus bikin orang lebih tertarik klik.

    Contoh meta description

    <meta name="description" content="Portofolio Muhammad Affizar: proyek Bootstrap, UI rapi, SEO-friendly, dan responsif. Lihat studi kasus dan kontak.">

    3. Struktur Heading: 1 H1, Baru H2/H3

    Heading itu kerangka bacaan. Idealnya hanya ada 1 H1 (judul utama), lalu pecah section besar pakai H2 dan detailnya pakai H3.

    Contoh struktur heading

    <h1>Portofolio Web Developer</h1>
<h2>Proyek Terbaru</h2>
<h3>Studi Kasus: Landing Page</h3>
<h2>Keahlian</h2>

    4. Internal Link: Arahkan User + Bot

    Internal link membantu mesin pencari memahami hubungan antar halaman, sekaligus membantu user eksplor konten. Di portofolio, ini penting untuk mengarahkan ke proyek, artikel terkait, dan kontak.

    Contoh internal link yang bagus

    <a href="artikel-seo-technical.html">Baca juga: SEO Technical Dasar</a>

    Aturan sederhana

    • Pakai anchor text yang jelas (jangan “klik di sini”)
    • Link ke halaman relevan (bukan asal banyak)
    • Pastikan tidak ada link rusak (404)

    5. Gambar: Alt Text + Nama File

    Alt text membantu aksesibilitas dan memberi konteks mesin pencari. Buat deskriptif, singkat, dan relevan. Jangan keyword stuffing.

    Contoh alt yang aman

    <img src="img/proyek-landing-page.webp" alt="Tampilan landing page Bootstrap untuk bisnis kuliner" loading="lazy">

    6. Schema: Bonus Sinyal (FAQ / Artikel)

    Schema (JSON-LD) membantu mesin pencari memahami tipe konten. Untuk artikel portofolio, yang paling gampang dipakai adalah BlogPosting dan FAQPage.

    Checklist schema

    • Judul + deskripsi sesuai konten
    • Author konsisten
    • Tanggal publish/modifikasi benar

    7. Checklist SEO On-Page Cepat

    • Title unik per halaman
    • Description jelas & menarik
    • 1 H1 + H2/H3 rapi
    • Internal link relevan
    • Alt image deskriptif
    • Open Graph untuk share sosial
    • Schema (Artikel + FAQ) jika cocok

    FAQ

    On-page fokus pada konten dan struktur di halaman (judul, deskripsi, heading, internal link, alt gambar, schema). Technical fokus pada crawl/indexing dan performa (sitemap, robots, Core Web Vitals, caching).

    Biasanya bukan faktor ranking langsung, tapi sangat mempengaruhi CTR (klik) di hasil pencarian. Description yang jelas dan menarik bikin orang lebih mau klik.

    Gunakan 1 H1 untuk judul utama. Pecah konten dengan H2 untuk bagian besar, dan H3 untuk subbagian. Yang penting strukturnya jelas dan nyaman dibaca.

    Kesimpulan

    SEO on-page itu pondasi paling cepat dirapikan: title & description yang kuat, heading terstruktur, internal link relevan, alt image rapi, dan schema bila perlu. Kalau ini konsisten, halaman portofolio kamu akan lebih “kebaca” dan lebih menarik diklik.
